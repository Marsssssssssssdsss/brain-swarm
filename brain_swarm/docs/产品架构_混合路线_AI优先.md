## 一、产品定义

### 一句话描述
> **NeuroResonator — 你的 AI 脑电共振器**

实时读取你的脑电波 → AI 识别专注/放松/冥想状态 → 精准闭环电刺激反馈 → 持续学习并优化。

| 场景 | 今天没有它 | 有了它 |
|------|-----------|--------|
| 工作专注 | 喝咖啡 + 番茄钟，效果不可控 | AI 检测到你分心 → 自动 tDCS 提升前额叶活跃度，你根本不知道它在工作 |

| 学习提升 | 不知道自己的注意力规律 | 连续记录 3 个月，AI 告诉你："你周二上午 9 点注意力最好，下午 3 点最差。建议把最难的工作安排在上午，下午做机械性工作。" |
| 睡眠 | 戴 Apple Watch 测个大概 | EEG 精准识别睡眠阶段，在你进入深睡期前自动关闭环境干扰 |

```
│  │  ├─ ADS1299 — 8ch EEG @250Hz, 24bit, 1μV         │       │
│  │  │  ├─ Fp1/Fp2 (前额专注)                         │       │
│  │  │  ├─ C3/C4 (运动皮层 mu 节律)                    │       │
│  │  │  └─ REF/GND (参考/地)                          │       │
│  │  ├─ BLE 5.4 — 手机 App 数据同步                    │       │

│  └──────────────┬────────────────────────────────────┘       │
│                 │ UART (115200, 带 CRC 校验)                  │
│  ┌──────────────▼────────────────────────────────────┐       │
│  │  AI 层 (ESP32-S3)                                 │       │
│  │  ├─ CMSIS-DSP FFT (512点, Hanning窗, 5频带)       │       │
│  │  ├─ TensorFlow Lite Micro (INT8, ~4KB, <50ms)    │       │
│  │  ├─ Heuristic 备选分类器 (TFLite 失效时降级)        │       │
│  │  ├─ 闭环规则引擎 (专注/放松 → tDCS 参数)           │       │
│  │  └─ WiFi 6 — OTA 固件更新 + 云端数据上传           │       │
│  └──────────────┬────────────────────────────────────┘       │
│                 │ SPI                                        │
│  ┌──────────────▼────────────────────────────────────┐       │
│  │  刺激层 (DAC8562 + OPA2189 Howland)               │       │
│  │  ├─ tDCS: 0~±2mA, 0.1μA 步进                    │       │
│  │  ├─ tACS: 0~1kHz, 可编程频率                      │       │
│  │  ├─ Ramp: 0.5mA/s 安全限速                        │       │
│  │  └─ 安全: 光耦隔离 + 过流保护 + 阻抗检测           │       │
│  └──────────────────────────────────────────────────┘       │
│                                                               │
│  ┌───────────────────────────────────────────────────┐       │
│  │  电源层 (BQ25120 + 402030 LiPo 400mAh)            │       │
│  │  ├─ 续航: ~4.6h 闭环 / ~8h 监测                     │       │
│  │  └─ 充电: USB-C, 1.5h 满                           │       │
│  └──────────────────────────────────────────────────┘       │
│                                                               │
│  ┌───────────────────────────────────────────────────┐       │
│  │  电极层 (干式梳齿 Ag-AgCl)                         │       │
│  │  ├─ 6ch: Fp1, Fp2, C3, C4, REF, GND              │       │
│  │  └─ 刺激: F3(阳极) + Fp2(阴极)                    │       │
│  └──────────────────────────────────────────────────┘       │
└──────────────────────────────────────────────────────────────┘
```

### 固件架构

```
nRF5340 (Zephyr RTOS, 传感器域):
  Thread 1: DRDY ISR @250Hz → SPI DMA → 环形缓冲
  Thread 2: 1Hz 打包 EEG → UART → ESP32-S3
  Thread 3: UART Rx → 解析闭环命令 → DAC 控制
  Thread 4: BLE 通知 @1Hz → 手机 App
  Thread 5: 安全监测 @2Hz → 阻抗/过流/超时

ESP32-S3 (ESP-IDF, AI域):
  Task 1 (Core 1): 1Hz 主循环
    → UART Rx → FFT → TFLite → 闭环规则 → UART Tx → Light Sleep
  Task 2 (Core 0): WiFi 后台
    → 每天一次 OTA 检查 → 云同步(夜间充电时)
  Task 3 (Core 0): 日志缓冲 → SD/Flash 写入
```

## 四、AI 层详细设计

### 4.1 设备端 AI（TFLite Micro, ESP32-S3）

```
Input: 4ch × 256 samples (1s @250Hz) → FFT → 20 band powers

On-device Model:
  ┌─ Conv1D(8 filters, kernel=16) ── BatchNorm ── ReLU ── GAP ──┐
  │                                                                │
  └───────────────────────────────────────────────────────────────┘
  ┌─ Dense(16) ── ReLU ── Dropout(0.2) ── Dense(8) ── Softmax ──┐
  │ 输出: {DROWSY, RELAXED, CALM, ATTENTIVE, FOCUSED,            │
  │        DEEP_FOCUS, HYPERFOCUS, NOISE}                        │
  └──────────────────────────────────────────────────────────────┘

大小: ~4KB (INT8 量化)
推理时间: <50ms (ESP32-S3 向量扩展)
RAM 占用: ~8KB 张量竞技场
更新: 每次 OTA 固件更新时附带新模型
```

### 4.3 产品路线图的 AI 叙事

> **硬件驱动的平台路径：所有人都说"生态不够"——先做好硬件，生态自然来了。**
>
> **我们做 EEG+tDCS，所有人说"脑电太 noisy，tDCS 效果不明确"——我们不等信号质量完美，先出货收集真实数据，AI 从数据中学习，模型越用越准。**
